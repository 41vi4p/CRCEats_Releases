## v1.3.1 — Smart Scheduling, Order Auto-Expiry & Canteen Status

**Release Date:** May 28, 2026

### ✨ New Features

**Auto-Expiring Orders** 🕐
- ✅ Unpaid orders automatically expire after 20 minutes
- ✅ Powered by Firebase Cloud Functions with zero polling
- ✅ Idempotent expiry handler — safe to retry
- ✅ Added `"Expired"` status to order lifecycle
- ✅ Seamless integration with existing order filtering

**Real-Time Canteen Status** 🏪
- ✅ Live canteen open/closed status from Firestore
- ✅ Red banner displays when canteen is closed
- ✅ Checkout automatically blocked during closed hours
- ✅ Managed by CRCEats-CS staff app
- ✅ Defaults to `open` for reliability

**Scheduled Order Time Restrictions** 📅
- ✅ Orders limited to **Mon–Sat, 10:00 AM – 4:00 PM** (canteen operating hours)
- ✅ Smart date picker — automatically skips Sundays
- ✅ Smart time picker — clamps selection to operating hours
- ✅ Default scheduled time set to next available canteen slot
- ✅ Validation prevents checkout outside operating hours

### 🔧 Improvements & Fixes

**Cart Page**
- ✅ Fixed stale unavailable-item warning that persisted after removal
- ✅ Replaced static `useEffect` with reactive `useMemo` — responds to real-time changes
- ✅ Auto-dismissing alert when unavailable items are detected
- ✅ Cleaner checkout handling with derived state

**Order Details Page**
- ✅ Fixed critical `zoneQRPages is not defined` crash
- ✅ Proper multi-zone QR code support with per-zone pages
- ✅ Zone-based sorting with itemized breakdown
- ✅ Fallback for legacy single-zone orders
- ✅ Improved stability and error handling

### 🏗️ Technical Highlights

**Cloud Functions Architecture** ☁️
- ✅ New `functions/` directory with TypeScript-based Cloud Functions
- ✅ `scheduleOrderExpiry` — Firestore trigger on order creation
- ✅ `expireOrder` — Cloud Task handler (20-minute delay)
- ✅ Zero database polling — efficient and scalable
- ✅ Full Firebase integration in `firebase.json`

**State Management**
- ✅ Added `Expired` to order status union types
- ✅ Updated `FirestoreService` with canteen status queries
- ✅ Enhanced `CanteenContext` with real-time status subscription
- ✅ Reactive state updates using `useMemo` for performance
